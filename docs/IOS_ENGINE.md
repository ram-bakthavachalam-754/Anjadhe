## The invariant

There are two renderers — `spec-renderer.js` (Mac, JS) and this SwiftUI engine
(iOS) — bound by **one contract** (`js/core/app-spec.js`) and **one conformance
corpus** (`tests/spec/corpus.json`). A component is "shipped" only when both
engines pass the same corpus. The SwiftUI engine is not a creative reimplementation;
it is a port that must match the reference renderer's behavior case-for-case.

**Source of truth, in priority order:**
1. `tests/spec/catalog.json` — the authoritative component list + `specVersion` +
   per-component `since`. Generated by `npm run catalog:export`; drift-checked by
   `npm test`. The engine declares the max `specVersion` it supports.
2. `tests/spec/corpus.json` — language-neutral cases (valid + invalid specs with
   expected validation errors). The iOS test target MUST run these.
3. `js/core/spec-renderer.js` — reference rendering/behavior for every component.
4. `js/core/app-spec.js` — the validator to port (`AppSpec.validate`).

## Inputs the engine consumes

A user app is a folder synced from the Mac (`userappsrc-<dir>` records, see
`js/core/user-apps-sync.js`):
- `manifest.json` — validated by the `AppManifest` port. **Only sync/run apps
  whose `manifest.entry === "app.spec.json"`** (portable). Code apps (`app.js`)
  are Mac-only — `AppManifest.portabilityOf` is the rule; the iOS sync filters to
  `portable`.
- `app.spec.json` — the document to render: `{ specVersion, title?, components: [] }`.
- `app.css` — ignored on iOS (the native engine owns styling; match the Mac theme).

## Data model

All app data is **scoped per app** (the Mac SDK's `Anjadhe.storageFor(appId)`),
a key→value store synced via the encrypted journal as `app_userapp-<id>`. The
engine reads/writes the same keys so data is identical across devices:

- `records:<collection>` → an **array of record objects**. Every record has a
  string `id` and ISO-8601 `createdAt`; other fields are user/spec data.
- `__spec_detail` → `{ collection, id }` — which record (if any) is open in a
  `record_list` detail view. Reserved; survives re-render.
- `__spec_tabs:<id-or-labels>` → active tab index for a `tabs` component.

Record id generation must be collision-resistant but format-free; the Mac uses
`r_<base36 time>_<base36 rand>`. Match the *shape* (unique string), not the exact
bytes.

## Validation (port of `AppSpec.validate`)

Port `js/core/app-spec.js` verbatim — same caps (`MAX_COMPONENTS 100`,
`MAX_DEPTH 4`), same per-component checks, same cross-check (every computed
aggregation's collection must be declared by a `form`/`record_list`/`lookup`, or
by a `button` action verb that names it, or by a `chart` grouping — see
`declaresCollection` semantics in the registry). The iOS unit tests assert each
`corpus.json` case validates with the same `ok` and the same error substrings.

**Catalog-version negotiation:** the engine supports up to some `specVersion` and
a set of component `type`s from `catalog.json`. If a synced spec uses a component
whose `since` exceeds what this build ships, render a clear "Update the app to see
this" placeholder for *that component only* — never fail the whole app, never
crash.

## Bindings / computed values

A **computed aggregation** is `{ count|sum|avg|min|max: "<collection>", field?, where? }`:
- resolve `records:<collection>`, apply `where` (exact field-equality filter),
- `count` → element count; `sum/avg/min/max` → over numeric `field`
  (non-numeric values ignored; empty → 0; avg of empty → 0).
- Display: integers as-is, else round to 2 decimals (`spec-renderer._aggregate`).

**showWhen** is `{ ...aggregation, op: gt|gte|lt|lte|eq|ne, value: <number> }` on
*any* component: evaluate the aggregation, compare to `value`; if false, omit the
component entirely. Unknown op → render (fail open).

## Action model (port of `_runAction`)

A `button.action` is exactly one bounded verb:
- `navigate` → open another app by `app` id (host navigation).
- `open_url` → open `url` externally (Safari/`UIApplication.open`).
- `add_record` → append `{ id, createdAt, ...values }` to `records:<collection>`.
- `clear_collection` → set `records:<collection>` to `[]`.
- `set_field` → on a **singleton** record (first record of the collection, created
  if empty), set `record[field] = value`.
- `increment` → on the singleton, `record[field] = (Number(record[field]) || 0) + (by ?? 1)`.

Mutating verbs save then re-render; `navigate`/`open_url` leave the view. The
singleton pattern is what makes counters/toggles work without a visible "add"
step — match `_singleton` exactly.

## Component rendering map (all 23 — match `spec-renderer.js`)

Treat all spec strings as untrusted text (SwiftUI `Text` is safe by default).
Theme: monochrome, `Text`/`Color.primary`-equivalents; semantic colors only for
`tone` (success `#16a34a`, warning `#d97706`, danger `#dc2626`).

| type | SwiftUI mapping / notes |
|---|---|
| `paragraph` | `Text(text)` |
| `section` | titled `VStack` (`title` as a section header); nests `components` |
| `divider` | `Divider()` |
| `card` | bordered/rounded container with optional title; nests `components` |
| `columns` | `LazyVGrid` with `count` (2–4) columns; 1 column on compact width |
| `tabs` | segmented control / tab bar; active index in `__spec_tabs:<key>`; render active tab's `components` |
| `summary_grid` | grid of value/label cards; value is scalar or computed |
| `list` | bulleted/numbered `List`/`VStack` of strings |
| `table` | header row + rows of cells |
| `form` | fields → inputs (text/textarea/number/date/checkbox/select); submit appends a record |
| `record_list` | rows from `records:<collection>`; `sort`, `allowDelete` (swipe), `editFields` (inline edit), `statusField` (tap-cycle chip), `detail` (push a detail view; optional one-time API enrichment via `detail.source`) |
| `lookup` | search field → debounced fetch of `source.url` (`{query}`), pick maps `fields` into a new record |
| `progress` | linear progress; value/max scalar or computed |
| `stat` | large value + label + optional caption |
| `badge` | small chip; `tone` → semantic color |
| `key_value` | label/value rows; value scalar or computed |
| `gauge` | radial progress — `Circle().trim(...)` (the Mac uses a conic-gradient ring) |
| `timeline` | vertical list with leading dots + connector line; label/time/detail |
| `button` | `Button(label)` running the action; `tone` styles it |
| `chart` | `bar`/`line`/`pie`/`area` via SwiftUI `Shape`/`Path` or Swift Charts; `data` is static points or a `{collection, groupBy, agg, field, where}` grouping (`_resolveChartData`) |
| `sparkline` | tiny inline line over `[number]` or a `{collection, field}` series |
| `image` | `AsyncImage(url)` + optional caption (http(s) only for now) |
| `icon` | named icon → **SF Symbol** (see map below) + optional label |

### Icon name → SF Symbol (mirror `ICON_GLYPHS`)

`star`→`star.fill`, `heart`→`heart.fill`, `check`→`checkmark`, `x`→`xmark`,
`home`→`house.fill`, `calendar`→`calendar`, `clock`→`clock.fill`, `flag`→`flag.fill`,
`bell`→`bell.fill`, `bolt`→`bolt.fill`, `book`→`book.fill`, `plus`→`plus`,
`arrow-up`→`arrow.up`, `arrow-down`→`arrow.down`. Keep this list = `AppSpec.ICONS`.

## Conformance the iOS engine must pass

1. **Validation parity:** run every `tests/spec/corpus.json` case through the
   ported validator; assert `ok` and error substrings match. This is the gate.
2. **Behavior parity (recommended):** mirror `scripts/spec-render-smoke.js` —
   aggregations compute the same numbers, `showWhen` hides/shows identically,
   action verbs mutate storage identically, chart grouping buckets identically.
3. **Catalog:** assert the engine's supported component set ⊇ every `type` in
   `catalog.json` at its `since ≤ supportedSpecVersion`, or has an explicit
   placeholder path for the rest.

## Out of scope for v1 (tracked in PLATFORM.md)

- **Local bundled assets** (`assets/` + the `anjadhe-app://` scheme) — media is
  http(s) URL only until that infra lands on both sides.
- **Live timer/stopwatch** components — need an interval lifecycle the engines
  don't model yet; still Mac-only JS escape-hatch territory.
- **audio / static-map** components — wait on the assets work.
